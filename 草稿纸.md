# git要求

| 标识       | 说明                 | 解释                           |
| -------- | ------------------ | ---------------------------- |
| init     | 初始化    项目初始化、模块初始化 |                              |
| feat     | 新功能    新增业务功能、新增接口 |                              |
| fix      | 缺陷修复               | 修复Bug、修复异常                   |
| test     | 测试代码               | 新增或修改测试用例代码                  |
| docs     | 文档变更               | 修改README、注释、API文档            |
| style    | 代码格式               | 代码格式调整（对齐、分号、空格等），未涉及功能修改    |
| refactor | 代码重构               | 重构代码，未涉及功能或缺陷修复              |
| perf     | 性能优化               | 提升性能、优化算法                    |
| build    | 构建调整               | 构建工具或依赖管理调整（pom.xml、gradle等） |
| ci       | CI/CD              | 自动化构建脚本的变更                   |
| chore    | 其他变更               | 其它非代码的变更（配置文件、脚本等）           |
| revert   | 版本回退               | 回退到之前的版本                     |

例如：

```
git commit -m "refactor: 重构算法匹配相关代码"
```
